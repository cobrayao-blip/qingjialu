# 民俗知识图谱：探索模式与工作台

本文说明主站「图谱」模块的产品能力、界面分区、筛选逻辑、与后端 API 的对应关系，便于研发、运营与内容编校对照实现。

---

## 1. 概述

### 1.1 定位

- **民俗知识图谱**将《清嘉录》相关的时令、活动、人物、地点、概念、体验等整理为 **实体（entities）** 与 **关系（relations）** 的规范结构（canonical graph）。
- 主站 **「图谱」** 标签页（`tab=graph`）提供两种界面：
  - **探索**：力导向图可视化（组件 `FolkloreGraph`），适合空间浏览、多视图切换与从节点跳邻居。
  - **工作台**：列表化研究界面（组件 `ResearchWorkbench`），适合复合筛选、研究篮子、批量导出与「待考」关系治理。

### 1.2 数据落盘与加载

- 权威数据文件：`server/data/folklore-graph.v1.json`（相对仓库根目录）。
- 服务端启动后通过内存缓存读取；规范化逻辑见 `server/services/folkloreGraphRepository.ts`（含 `evidence` 字段缺省时的「待考」补齐规则）。
- 前端数据模型类型定义：`src/graph/folkloreGraphModel.ts`。

### 1.3 与《清嘉录》原文

- 图谱中的文献节点与引用可与全局 **原文抽屉** 联动：从节点侧栏点击「查看原文 / 打开原文」会调用 `onOpenQjlSection`，按 `sectionId` 或章节标题匹配打开对应小节并支持引文高亮。

## 2. 数据模型摘要

### 2.1 实体类型 `EntityType`

| 值 | 含义 |
|----|------|
| `time` | 时令（月份等） |
| `practice` | 活动 / 习俗实践 |
| `actor` | 人物 |
| `place` | 地点 |
| `artifact` | 物件 |
| `source` | 文献 |
| `concept` | 概念 |
| `experience` | 体验（含感官维度叙事） |

实体常见字段：`id`、`label`、`type`、`description`、`conceptTags`、`experienceTags`、`meta` 等。

### 2.2 关系类型 `RelationType`

| 值 | 前端展示（探索侧栏等） |
|----|------------------------|
| `occurs_in` | 发生于 / 月令 |
| `occurs_at` | 发生在 / 地点 |
| `performed_by` | 参与者 |
| `uses` | 使用 / 器物 |
| `documented_in` | 文献依据 |
| `related_to` | 相关 |
| `symbolizes` | 象征 |
| `regulates` | 规范 |
| `evokes` | 唤起 |
| `associated_with` | 关联 |

### 2.3 关系证据 `evidence` 与统计口径

- 加载图时，**非 `related_to`** 的关系若缺少 `evidence` 字符串，会被规范为 **`待考`**（表示仍需补文献或说明）。
- **证据健康度**（`/api/graph/evidence-metrics` 与 `GET /api/graph` 内嵌的 `evidence`）大致规则：
  - `requiredRelations`：`relationType !== 'related_to'` 的关系条数。
  - `withEvidence`：`evidence` 非空且不等于 `待考` 的条数。
  - `pending`：`requiredRelations - withEvidence`。
  - `coverage`：`withEvidence / requiredRelations`（无必填关系时为 `1`）。

工作台中的「待考关系清单」与导出任务单，对应 **`evidence === '待考'` 且非 `related_to`** 的关系。

## 3. 前端入口与 URL

### 3.1 主导航

- 进入 **图谱** 页：`tab=graph`（应用内切换主导航时会 `replaceState` 写入）。

### 3.2 探索 / 工作台

- Query：`graphMode=explore` 或 `graphMode=workbench`。
- 切换「探索 / 工作台」按钮会同时设置 `tab=graph` 与 `graphMode`。
- 离开图谱页到其他 tab 时，会 **删除** `graphMode` 参数，避免误带到其他模块。
- **兼容**：旧链接 `tab=workbench` 会被解析为图谱页且模式为工作台。

### 3.3 月份与图谱子图

- 图谱页左侧 **月份** 列表会设置「当前图谱月份」并传给探索组件；文案约定：**月令、地景、叙事** 三种预设视图在探索模式下 **统一按该月份抽取子图**（见下文 API `month` 参数）。
- 工作台在 `month` 属性变化时会 **自动把「月份筛选」设为该值**，用于列表过滤（与探索共用左侧栏所选月份语义）。

## 4. 探索模式（`FolkloreGraph`）

路径：`src/components/FolkloreGraph.tsx`。

### 4.1 画布交互

- **D3 力导向布局**：节点可拖拽；**滚轮缩放**；**拖动画布**平移。
- 底部提示：滚轮缩放 · 拖拽移动 · 点击查看详情。

### 4.2 数据请求与降级

- 使用 **`GET /api/graph/subgraph`**（封装为 `getFolkloreSubgraph`），请求参数包含：`preset`、`month`、`relationTypes`、`onlyWithSources`。
- 请求前 **200ms 防抖**；相同 query key 会跳过重复请求。
- 若接口失败或返回无效：使用前端 **内置图 + 本地切片** 作为 fallback（内置图在模型中可为空壳；生产环境以服务端 JSON 为准）。

### 4.3 三种视图预设 `GraphViewPreset`

| UI 名称 | `preset` 值 | 主要实体切片（默认，可被 query 覆盖） | 默认关系切片（可被前端关系筛选覆盖） |
|---------|-------------|----------------------------------------|----------------------------------------|
| 月令 | `month_custom_role` | `time`、`practice`、`actor`、`source` | `occurs_in`、`related_to`、`performed_by`、`documented_in` |
| 地景 | `time_place_practice` | `time`、`place`、`practice`、`source` | `occurs_in`、`occurs_at`、`documented_in` |
| 叙事 | `narrative_practice_concept_experience` | `practice`、`concept`、`experience`、`source` | `symbolizes`、`regulates`、`evokes`、`associated_with`、`documented_in` |

预设默认切片由 `getPresetSliceOptions` 定义；服务端子图接口对 **未传** `entityTypes` / `relationTypes` 时使用预设默认值，若前端传了 CSV 则覆盖。

### 4.4 月份子图语义（服务端与模型一致）

1. 在 **全图** 中查找 `type === 'time'` 且 `label === month` 的实体。
2. 通过 `occurs_in`（source 为月份、target 为活动）收集当月 **活动** id。
3. 子图包含：该月份节点、上述活动、以及与这些活动 **直接相连** 一跳的邻居实体与它们之间的边（在后续 `sliceCanonicalGraph` 中再按实体类型/关系类型收缩）。

实现参考：`sliceCanonicalGraphByMonthLabel`（`src/graph/folkloreGraphModel.ts`）。

### 4.5 悬浮控制区

- **显示/隐藏控件**：避免遮挡画布。
- **拖动控件**：整块控制面板可拖动（支持触摸）。
- **搜索**：
  - 有关键词：建议列表合并「当前视图内匹配」与「全图实体匹配」，去重后最多 5 条；支持键盘上下与 Enter。
  - 无关键词：展示最近访问节点（最多 5 条）。
- **图谱筛选**：随当前预设切换子选项（如月令：月份 / 习俗 / 人物；地景：时令 / 地点 / 活动；叙事：活动 / 概念 / 体验）。用于 **隐藏某类投影节点**；若当前选中节点被筛掉，会自动清空选中。
- **高级筛选**：
  - **仅显示有文献依据的节点**：保留至少有一条 `documented_in` 指向 `source` 可解析出引用的实体（与 `listSourceCitationsForEntity` 一致）。
  - **关系类型**：多选；影响画布上的边；可与预设默认关系取交集逻辑配合（以前端传入 `relationTypes` 为准）。

### 4.6 节点详情侧栏

- **实体概要**：类型徽章、标题。
- **详细解析**：描述；支持 **浏览器语音朗读**（`speechSynthesis`，中文）。
- **结构化字段**（若有）：时间、地点、职责等。
- **概念意义 / 感官体验**：标签。
- **关联关系**：
  - 列出与当前实体 **全图范围内** 的直接边（不限于当前画布可见边）。
  - 可点击在图中定位邻居（「在图中查看」）；画布高亮当前选中。
- **文献依据**：标题、引句；可打开原文抽屉。

### 4.7 与侧栏「关系类型」的说明文案

画布上的边受「关系类型」筛选；侧栏邻居列表为 **全图语义关系**。避免用户误以为两者不一致是 bug。

## 5. 工作台（`ResearchWorkbench`）

路径：`src/components/ResearchWorkbench.tsx`。

### 5.1 数据加载

- **全图**：`GET /api/graph` → `getFolkloreGraph()`，仅使用响应中的 `entities` / `relations`。
- **证据指标**：`GET /api/graph/evidence-metrics` → `getFolkloreEvidenceMetrics()`（失败时忽略，不阻塞主列表）。

### 5.2 布局（大屏 `xl`）

三列栅格：**左 3 / 中 5 / 右 4**；小屏纵向堆叠，各块有最大高度与独立滚动。

### 5.3 左栏：研究筛选

| 控件 | 行为 |
|------|------|
| 查询方案下拉 | 选择已保存方案并 **应用** 其筛选字段 |
| 保存查询 | 将当前筛选写入 **当前激活** 方案 id |
| 另存为 | `prompt` 输入名称，新建方案并激活 |
| 恢复查询 | 用当前方案快照覆盖 UI（撤销未保存的手改） |
| 清空 | 关键词、月份、实体类型全选、标签清空、关系类型恢复为全部、排序恢复默认 |
| 删除 / 重命名 | 维护方案列表 |
| 关键词 | 在 `label` + `description` 中不区分大小写子串匹配 |
| 仅看待考关系相关实体 | 仅保留出现在「待考关系」两端的实体 |
| 关系类型 | 多选；实体必须在 **至少一条选中类型** 的边上出现 |
| 月份 | 「全部月份」或具体月；逻辑见下 |
| 实体类型 | 多选 toggles |
| 概念标签 / 感官标签 | 多选；对同一组为 **AND**（所选标签必须全部命中 `listNarrativeTagsForEntity` 合并结果） |

**月份过滤逻辑（与探索子图不同，更宽）**

- 若选了具体月份：找到对应 `time` 实体，经 `occurs_in` 得到当月 **活动 id 集合**。
- **时令**：仅保留 `label === monthFilter` 的月份节点。
- **活动**：仅保留属于上述集合的活动。
- **其他类型**：保留与这些活动通过 **任意关系** 在图中 **连通** 的实体（通过遍历 `relations` 判断是否存在经活动关联的路径）。

### 5.4 中栏：检索结果

- **排序**：按文献引用条数降序（默认）、按名称、按类型再按名称。
- **研究篮子**：对实体打标集合；可展开面板、导出篮子、清空、单项移除。
- **列表项**：展示名称、类型、文献条数；点击进入右侧详情；**加入篮子** 独立按钮（阻止冒泡）。

### 5.5 右栏：证据与导出

| 功能 | 说明 |
|------|------|
| Markdown / JSON 导出 | 默认导出对象为 **当前筛选结果**；若篮子非空则 **优先导出篮子** |
| 证据健康度 | 展示覆盖率、待考条数、总关系数（来自 metrics） |
| 待考关系清单 | 列出部分待考边；点击一条可将 **source** 设为当前选中实体 |
| 导出任务单 | 生成 `pending-relations-todo.md`：按关系类型分组，含 id 与补证提示（sectionId + quote） |
| 选中实体详情 | 描述、概念/感官标签、文献列表（可打开原文）、**叙事路径**（活动→概念→体验，最多约 20 条） |

### 5.6 查询方案持久化

- 列表 key：`qingjialu.research.workbench.queries.v1`
- 旧版单条 key（兼容读取）：`qingjialu.research.workbench.query.v1`
- 方案内容包含：关键词、月份、实体类型集合、概念/感官标签、排序模式等（**不含**关系类型多选——关系筛选为会话级；若需写入可后续迭代）。

### 5.7 导出文件命名

- 筛选结果：`research-workbench.md` / `research-workbench.json`
- 篮子：`research-basket.md` / `research-basket.json`
- 待考任务单：`pending-relations-todo.md`

## 6. HTTP API 摘要

### 6.1 公开只读

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/graph` | 返回完整 canonical 图 + `meta` + `evidence` |
| `GET` | `/api/graph/evidence-metrics` | 返回 `meta` + `evidence`（体量小于全图） |
| `GET` | `/api/graph/subgraph` | 返回子图，见下表 Query |

**`/api/graph/subgraph` Query 参数**

| 参数 | 说明 |
|------|------|
| `preset` | `month_custom_role` \| `time_place_practice` \| `narrative_practice_concept_experience`；无效则视为未指定预设 |
| `month` | 月令文字，如 `正月`；与全图中的 `time.label` 对齐 |
| `entityTypes` | 逗号分隔，覆盖预设默认实体类型；非法 token 过滤 |
| `relationTypes` | 逗号分隔，覆盖预设默认关系类型 |
| `onlyWithSources` | `1` 或 `true` 为真：仅保留存在聚合文献引用的实体及其之间的边 |

处理顺序（与路由实现一致）：**全图 → 按月份切片 → `sliceCanonicalGraph`（实体类型 + 关系类型）→ `onlyWithSources` 再收缩**。

### 6.2 管理端（需 `admin` / `editor` 角色）

与编校流程相关，图谱页普通用户不使用：

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/graph/reload` | 从磁盘重新加载 `folklore-graph.v1.json` |
| `POST` | `/api/graph/admin/merge-from-qjl` | 按《清嘉录》小节合并缺口；`dryRun` 仅统计 |
| `GET` | `/api/graph/admin/drafts` | 草稿列表 |
| `POST` | `/api/graph/admin/drafts` | 保存草稿（title 必填；可选 body.payload） |
| `POST` | `/api/graph/admin/publish-draft/:id` | 发布草稿到磁盘并热重载 |
| `DELETE` | `/api/graph/admin/drafts/:id` | 删除草稿 |

详见管理端 UI：`src/components/AdminApp.tsx` 中「民俗图谱」一节。
